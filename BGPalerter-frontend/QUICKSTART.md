## Configuration

### BGPalerter API Connection

The dashboard connects to BGPalerter via REST API (default: `http://127.0.0.1:8011`).

**If BGPalerter is on a different host/port:**

```bash
cd ~/BGPalerter/BGPalerter-frontend
nano .env

# Change this line:
BGPALERTER_API_URL=http://YOUR_BGPALERTER_HOST:PORT

# Restart dashboard
pm2 restart bgpalerter-dashboard
```

### Webhook Notifications

Configure in the dashboard UI:

1. Navigate to **Administration** page
2. Enable desired webhook (Teams/Slack/Discord)
3. Enter webhook URL
4. Set minimum severity level
5. Click **Save Settings**
6. Use **Test** button to verify

## Troubleshooting

### Dashboard shows "Disconnected"

**Problem**: Can't connect to BGPalerter API

**Solution**:
```bash
# 1. Verify BGPalerter is running
docker ps | grep bgpalerter
curl http://127.0.0.1:8011/status

# 2. Check dashboard logs
pm2 logs bgpalerter-dashboard | grep "BGPalerter"

# 3. Verify BGPALERTER_API_URL in .env
cd ~/BGPalerter/BGPalerter-frontend
cat .env | grep BGPALERTER_API_URL
```

### Dashboard won't start

**Problem**: PM2 shows "errored" status

**Solution**:
```bash
# Check logs for errors
pm2 logs bgpalerter-dashboard --lines 50

# Common fixes:
# - Database connection: verify DATABASE_URL in .env
# - Port conflict: change PORT in .env
# - Permissions: sudo chown -R $USER:$USER ~/BGPalerter/BGPalerter-frontend
```

### No alerts showing

**Problem**: Alert History page is empty

**Solution**:

This is normal if BGPalerter hasn't detected any anomalies yet. The dashboard displays alerts generated by BGPalerter.

To test:
1. Check BGPalerter logs: `docker logs bgpalerter --tail 100`
2. Verify monitors are active in dashboard Home page
3. Wait for BGPalerter to detect routing changes

## Production Deployment

For production use, set up HTTPS with Nginx:

```bash
# Install Nginx
sudo apt install -y nginx

# Install SSL certificate
sudo apt install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your-domain.com

# Configure reverse proxy (see DEPLOYMENT.md for details)
```
